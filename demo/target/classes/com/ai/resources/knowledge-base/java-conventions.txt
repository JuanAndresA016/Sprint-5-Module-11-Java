Package structure: com.ai.crud.entity, com.ai.crud.repository, com.ai.crud.service, com.ai.crud.controller

All entities must use @Entity and @Table(name = ...) annotations.
Use Long as the primary key type with @GeneratedValue(strategy = GenerationType.AUTO).
Repositories extend JpaRepository<Entity, Long>.
Service classes are annotated with @Service and @Transactional.
Controllers use @RestController and @RequestMapping("/api/v1/{entity}").

REST endpoint conventions:
- GET    /api/v1/{entity}       -> findAll()
- GET    /api/v1/{entity}/{id}  -> findById()
- POST   /api/v1/{entity}       -> create()
- PUT    /api/v1/{entity}/{id}  -> update()
- DELETE /api/v1/{entity}/{id}  -> delete()

Import conventions:
- import jakarta.persistence.*;
- import org.springframework.web.bind.annotation.*;
- import org.springframework.http.ResponseEntity;
- import org.springframework.stereotype.Service;
- import org.springframework.transaction.annotation.Transactional;
- import org.springframework.data.jpa.repository.JpaRepository;

Naming conventions:
- Entity class: PascalCase (e.g., ProductItem)
- Table name: snake_case (e.g., product_items)
- Repository: {Entity}Repository
- Service: {Entity}Service
- Controller: {Entity}Controller
